Help

About

This web application is typically used for adding meta data to a collection of multimedia files (such as images or movies). The meta data is called categories in this web application.

File Categories

The following meta data can be added to each file from the file collection:

  • File description
  • File date and time
  • Persons
  • Locations
  • Tags

The meta data above, called categories, can be used when searching for files or to document historic events represented by the files.

File Collection

FileDB does not modify any files in your file collection. When adding files they are read to extract date and time information when available.

Any directory structure can be used for sorting files in your existing collection. However, this layout has been proven to work very well:

root
 |- 2015
 |   |- 2015-06-12 - Biking
 |   |      |- file1.jpg
 |   |      |- file2.jpg
 |   |
 |   |- 2015-07-13 - Birthday party
 |          |- somefile.jpg
 |
 |- 2016
     |- 2016-01-05 - Something else
            |- anotherfile1.jpg
            |- anotherfile2.jpg

The root directory can be placed anywhere, see configuration chapter.

Configuration

Configuration parameters are set in a configuration file with Python format. Each parameter is documented in that configuration file (see config.py).

Backup

File catagories are written to a database which is stored in the file flaskr.db. Make regularly backups of this file to avoid losing any data in case of hardware, software or human mistake.

API

This web application fetches and updates data in the FileDB database via its API. The API can be used by other applications.
