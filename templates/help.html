Help

About

This web application is used for storing and presenting meta data for your multimedia files (such as images or movies).

The following meta data can be added to each file from the file collection:

  • File description
  • File date and time
  • Persons
  • Locations
  • Tags

The meta data above, called categories, can be used when searching for files or to document historic events represented by the files.

Installation

Read more about installation, configuration, backup and the HTTP API in the README (not available here).

Note that because FileDB extracts date and time information from added JPEG images, make sure that your cameras have the correct date and time set.

Categorization

When FileDB is up and running it is time to create the categories at the Edit Categories page. Afterwards files can be categorized at the Categorize Files page.

It is recommended to only categorize persons in pictures that are clearly visible in the foreground.

Notes About Your Files Collection

FileDB does not modify any files in your file collection. Files are read when importing (adding) files, viewing files or when reading Exif-data via the API.

Before importing your files, take a moment to think about the directory structure. Any directory structure can be used for sorting files in your existing collection. However, this layout has been proven to work very well:

root
 |- 2015
 |   |- 2015-06-12 - Biking
 |   |      |- file1.jpg
 |   |      |- file2.jpg
 |   |
 |   |- 2015-07-13 - Birthday party
 |          |- somefile.jpg
 |
 |- 2016
     |- 2016-01-05 - Something else
            |- anotherfile1.jpg
            |- anotherfile2.jpg

Files can have the same name in different directories.

File paths in FileDB uses UNIX style directory separators ('/'), mostly because that is how FileDB stores the paths internally.

If you rename or delete files that has been added to FileDB you need to remove them in FileDB as well (and possibly add them again). There is a tool for finding such files at the Add Files page
